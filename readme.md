# 涉县乡村公益数字大脑 (Smart Village Platform)

“涉县乡村公益数字大脑” 是一个为涉县量身定制的乡村数字化公益服务系统。该项目主旨涵盖：**智慧助农、乡村学堂、文化融合、数据赋能** 四大核心模块。系统采用前后端分离的架构，旨在为农户、留守儿童、游客（C端用户）以及村支书/管理团队（B/G端用户）提供一体化的数字平台解决方案。

## 🛠️ 技术栈选型架构

本项目划分为用户端小程序、管理端后台和核心服务端三层：

| 层级 | 平台/框架 | 核心技术 & 作用 |
| --- | --- | --- |
| **用户端 (To C)** | **微信小程序 (Uni-app / Vue)** | 面向农户、儿童、游客。<br>涉及：`<map>` 组件 (LBS服务)、`uni.getRecorderManager()`录音等。 |
| **管理端 (To B/G)**| **Vue 3 + Vite + Element Plus** | 面向村支书、超级管理员。<br>涉及：数据看板管理、应用审批、后台配置、ECharts 可视化。 |
| **服务端 (Backend)**| **Python (FastAPI)** | 提供轻量、高并发异步的 API 接口。<br>集成接口文档自动生成。 |
| **数据与分析库** | **MySQL**, `jieba`, `matplotlib` 等| MySQL 负责持久化存储 E-R 数据实体；Python 分析库负责长文本词频剖析、热力图生成。 |
| **外部依赖API** | **OpenWeather API** | 获取实时涉县气象数据，辅助辅助农业决策计算。 |

## 🚀 核心功能模块划分

### 模块一：智慧助农
- **节水决策计算器：** 后端调用气象 API，前端让农户选择作物（核桃/花椒）及生长期，算法输出是否需要灌溉的节水建议。
- **用水公平调度台：** 村民通过 C 端提交用水申请，村支书（管理员）在后台“用水调度审批”界面进行审批（通过/拒绝）。

### 模块二：乡村学堂 (教育赋能)
- **分龄推普与法治影院：** 提供儿童动画、职场普通话、老年防诈三大频道，观影后配合简单的知识问答（单选题计分制）。
- **留守儿童云陪伴：** 打造每日打卡流，孩子可上传作业照片或朗读语音，优秀内容将进入首页轮播。

### 模块三：文化融合 (LBS + UGC 体验)
- **红色有声地图：** 在小程序地图上标记一二九师纪念馆等红色地标，点击弹出后台流式传输的语音/视频讲解。
- **方言故事共创：** 游客与当地人利用小程序录制涉县方言音频并上传，交由管理员审核后全网发布。

### 模块四：数据赋能 (Admin 后台监控)
- **农业需求热力图：** 利用 `jieba` 中文分词工具，提取农户提问与搜索的高频词汇（如“黑腐病”、“干旱”），渲染出农业舆情热力走向。
- **数据可视化大屏：** 提取关键业务指标（如：总用户活跃度、视频完课率、今日待审批用水申请），利用 ECharts 生成多维度的图表展板。

## 🚦 当前项目进度与不足 (需关注)

本项目目前正处于快速迭代开发阶段，核心架构搭建与部分业务逻辑已经打通，但距离最终落地仍有以下不足需要攻克：

### 1. C端小程序开发未完成 (WIP)
目前代码库中包含了完备的**管理端前台 (Admin-Frontend)** 与 **Python FastAPI 服务端 (Backend)** 且已经实现互通，但**供普通用户使用的“微信小程序端 (user-mini-app)”目前仅搭好了骨架，具体的界面编写和接口联网代码尚未开发完毕**。

### 2. 整体程序尚未进行全链路测试 (Untested)
- 虽然管理端后台的“用水调度审批”等模块已经能够成功对接数据库进行修改，但由于 C 端输入口尚未完工，我们缺乏真实环境下的**全栈端到端测试 (End-to-End Testing)**。
- 涉及到高频词库解析 (`jieba`) 和地图 LBS 功能等复杂模块的联调还未在真机上验证是否存在性能或兼容性问题。

### 3. 未接入正式的系统登录与鉴权
目前管理系统与服务端的 API 还采用单机跑通的简单流程，未来需要将小程序的微信鉴权 (`OpenID`) 与管理端的 `Token` (如 JWT) 打通并限制非法请求。

## 📂 系统层级架构总结

整个系统按标准分为四层：
1. **表现层 (Presentation Layer):** 微信小程序交互层 + Web数据大屏管理端。
2. **业务逻辑层 (Business Service Layer):** 天气助农计算器、用水审批状态机、视频流分发、音频资源接收与发布审核。
3. **数据分析层 (Data Analysis Layer):** 词频统计需求挖掘引擎、Echarts 图表数据清洗流。
4. **数据持久层 (Data Layer):** MySQL 关系型存储（保存用户、资源、审批流）与本地/云端 OSS（存储图像和视频文件等二进制资源）。
