# 涉县乡村公益数字大脑 (Smart Village Platform)

“涉县乡村公益数字大脑” 是一个为涉县量身定制的乡村数字化公益服务系统。该项目主旨涵盖：**智慧助农、乡村学堂、文化融合、数据赋能** 四大核心模块。系统采用前后端分离的架构，旨在为农户、留守儿童、游客（C端用户）以及村支书/管理团队（B/G端用户）提供一体化的数字平台解决方案。

## 🛠️ 技术栈选型架构

本项目划分为 Web 前端（多角色门户）、管理端后台和核心服务端三层：

| 层级 | 平台/框架 | 核心技术 & 作用 |
| --- | --- | --- |
| **用户端 (To C)** | **React + Vite** | 面向农户、儿童、游客的 Web 门户。<br>涉及：多角色视图路由、现代 UI 组件交互、原生视频/录音 Web API 接入。 |
| **管理端 (To B/G)**| **React + Vite** | 面向村支书、管理员。<br>涉及：数据看板管理、应用审批、高德地图 API 景点配置、视听资源审核。 |
| **服务端 (Backend)**| **Python (FastAPI)** | 提供轻量、高并发异步的 API 接口。<br>集成接口文档自动生成。 |
| **数据与存储库** | **MySQL**, **SQLAlchemy**| MySQL 负责持久化存储 E-R 数据实体；使用 Local Storage 暂代 OSS 存储媒体文件。 |
| **外部生态接口 (API)**| **高德地图、和风天气**| **高德地图** (AMap)：支撑文化地标的定位与地图选点业务。<br>**和风天气** (QWeather)：提供气象数据支撑智慧助农模块的节水灌溉算法。|

## 🚀 核心功能模块划分 (开发进度)

### 模块一：智慧助农 (✅ 核心已完成)
- **前端门户：** 农户可以在农业大厅查看最新资讯、向管理员提交农事问题反馈（带图），并在线提交果园用水时长的申请。
- **后台审批：** 管理员/村支书在后台可一键通过或驳回农户的用水申请，并记录审核意见。

### 模块二：乡村学堂 (教育赋能) (✅ 核心已完成)
- **分龄推普与法治影院：** 面向农户与儿童的在线视频学习库。前端支持视频流式播放与观看记录打点。
- **管理端视频监控：** 管理员可上传新的教学视频，并在后台实时查看各视频的完播率与每个农户的实际观看时长进度。

### 模块三：文化融合 (LBS + UGC 体验) (✅ 全面完成)
- **红色地标网格：** 在文化页展示涉县一二九师纪念馆等红色地标列表。
- **方言故事共创：** 详情页支持**在线麦克风录音**和**本地音频上传**，故事精准绑定特定景点打卡。
- **景点与审核管理：** 管理后台集成了**Vue高德地图取点组件**，搜索关键词可直接在地图上定位新建景点。对用户上传的方言故事可在线试听后执行发布流或拒绝打回。

## 🚦 当前项目进度与不足 (待优化项)

本项目目前正处于快速迭代开发阶段，核心的 Web 前后双端业务流程已经打通，但在商业化和完美落地前，仍有以下工作计划：

### 1. 游客模式待开发 (Pending)
目前系统涵盖了“农户”、“志愿者”、“管理员”登录与各自生态链。免登录的“游客模式”功能树（如简单的打卡浏览、不保留记录的地图查看等）仍在排期研发中。

### 2. 多端小程序迁移待定 (Discussing)
项目初期计划包含了农户、志愿者、游客端的微信小程序接入（`Uni-app/Vue`）。目前 Web 版本的 C 端功能已成熟并经过验证，是否继续投入资源开发微信生态内的小程序版本将根据未来实际运营需求决定。

### 3. 系统细节与性能优化 (Bug Fixes)
- 视频大文件流式加载在低带宽下偶尔会出现卡顿，计划引入分片上传与断点续传机制优化。
- 图片与音频资源目前依赖本地 IO 存储，下一步需向云端 OSS 对象存储迁移以支持高并发。

### 4. 数据库文档滞后待更新 (Doc Update)
**今日进度：** 为支持将方言故事精准绑定至独立景点，我们在 `models.py` 和 MySQL 数据库的 `dialect_stories` 原表中强行侵入新增了 `landmark_id` 与 `reject_reason` 映射字段。
由于快速的敏捷迭代，原本最初定稿提交的 **E-R 数据流向图与部分纸质结构设计文件目前与实际代码库架构存在脱节，待近期内统一梳理更新。**

### 5. AI 普通话语音教辅模块 (TBD)
**正在调研/待定开发：** 计划在乡村学堂的推普板块中，接入**科大讯飞 (iFLYTEK) 语音评测 API**。该模块将允许农户和儿童上传自己的发音跟读录音，系统利用科大讯飞的大模型能力进行普通话发音纠正与打分，从而实现更具互动性的数字教学闭环。（*注：因涉及第三方计费 API 集成难度，该模块为弹性选做项*）
